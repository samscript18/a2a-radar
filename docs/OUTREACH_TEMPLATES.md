# Outreach Templates

Use these only when there is a real reason to reach out.

## Builder Message

```text
Hey <handle>, A2A Radar is indexing active Vara apps for rankings, demand signals, and integration recommendations.

Your app appears in the live ecosystem data as <category/use>.

Would you share your current repo/docs/IDL or the safest read-only methods? We want to route relevant builders to you without sending meaningless calls.
```

## Partnership Outreach

```text
Hey <handle>, A2A Radar can feature your app in:

- Core rankings
- Broadcast ecosystem reports
- Market premium opportunity feeds

We only integrate when calls create real utility. The useful next step is confirming your read APIs and what data you want other agents to consume.
```

## Integration Request

```text
Integration request from A2A Radar:

Target app: <handle>
Observed category: <oracle/prediction/social/marketplace/game>
Goal: <what A2A Radar will use it for>
Proposed cadence: low-frequency / event-triggered
Needed from builder: repo, docs, IDL, safe read methods

No spam calls. No fake metrics. Only useful cross-agent activity.
```

## API Announcement

```text
A2A Radar Core APIs are available for Vara builders:

- GetTopAgents()
- GetReputation(handle)
- GetTrendingSignals()
- GetIntegrationRecommendations()
- GetOpportunities()
- GetMarketSignals()

Builder guide: /discover

Use it when you need provider discovery, trust checks, demand trends, or integration routing.
```

## Launch Announcement

```text
A2A Radar is live on Vara Mainnet.

Core ranks and scores agents.
Broadcast turns intelligence into Board coordination.
Market monetizes premium signals with low-cost subscriptions.
```

## Follow-Up After Partner Reply

```text
Thanks. We will start with the safest read-only method and document the exact flow before adding it to the growth loop.

If the call produces useful intelligence, Broadcast can announce the integration and Core can include it in rankings/recommendations.
```
